# Quick start

This guide walks you through generating your first worklog, from listing projects to viewing the generated summary.

<Note>
  Make sure you've completed the [installation](/installation) steps before continuing.
</Note>

## Step 1: List your projects

First, verify your projects are configured correctly:

```bash
mcp-worklog.js list-projects
```

**Example output:**

```
Projects: [ 'crm-system', 'ecommerce-app' ]
activeProject: crm-system
```

This command reads your `~/.worklogs/projects.json` configuration and displays:
- All configured project names
- The currently active project

<Note>
  The `list-projects` command is implemented in `mcp-worklog.js:198-201`.
</Note>

## Step 2: Set the active project

If you want to generate a worklog for a different project, set it as active:

```bash
mcp-worklog.js set-active ecommerce-app
```

**Example output:**

```
activeProject set to ecommerce-app
```

This updates the `activeProject` field in your configuration file. The change persists across sessions.

<Warning>
  The project name must exist in your `projects.json` configuration. Use `list-projects` to see available options.
</Warning>

## Step 3: Generate your first worklog

Now generate a worklog for today's commits:

```bash
mcp-worklog.js generate
```

You'll see a spinner while MCP Worklog:
1. Extracts commit hashes from your Git repository
2. Collects diffs for each commit (up to 10 commits)
3. Sends the diffs to Gemini for summarization
4. Saves the formatted output

**Example output:**

```
✓ Generating worklog for ecommerce-app...
Saved /home/username/projects/ecommerce-app/worklogs/2026-03-03.md
```

## Using different date ranges

MCP Worklog supports flexible date parameters for generating worklogs:

<CodeGroup>
  ```bash Today (default)
  mcp-worklog.js generate
  # or explicitly
  mcp-worklog.js generate today
  ```

  ```bash Yesterday
  mcp-worklog.js generate yesterday
  ```

  ```bash Specific date
  mcp-worklog.js generate 2025-08-01
  ```
</CodeGroup>

The `parseSinceDate()` function converts these inputs to ISO timestamps for Git:

- `today` → Today at 00:00 (midnight)
- `yesterday` → Yesterday at 00:00
- `YYYY-MM-DD` → Specified date at 00:00

<Note>
  Date parsing is handled in `mcp-worklog.js:77-119`. All dates are normalized to midnight in your local timezone.
</Note>

## Understanding the generated worklog

Open the generated Markdown file to see your worklog:

```bash
cat ~/projects/ecommerce-app/worklogs/2026-03-03.md
```

**Example worklog structure:**

```markdown
# Work Log — 2026-03-03

## Developer Work Log
- Added user authentication flow [a79fc44] - auth.js, login.js, middleware.js
- Fixed memory leak in data fetcher [3b4c5d6] - fetcher.js, cache.js
- [refactor] Simplified error handling logic [7e8f9g0] - error-handler.js
- [docs] Updated API documentation [2c3d4e5] - README.md, api-docs.md

---

## Manager Summary
**Progress:** Completed authentication system and resolved performance issues
- User login and session management now fully functional in production
- Fixed critical memory issue affecting data loading, improving app responsiveness
```

The worklog includes:
- **Developer Work Log** - Technical bullets with commit hashes and file changes
- **Manager Summary** - Business-friendly progress overview

## How the generation works

Here's what happens when you run `generate`:

<Steps>
  <Step title="Load configuration">
    Reads `~/.worklogs/projects.json` to get the active project path:

    ```javascript
    function getActiveRepoPath() {
      const cfg = loadConfig();
      const name = cfg.activeProject;
      const repo = cfg.projects[name];
      return { name, repo, worklogFolderName: cfg.worklogFolderName || "worklogs" };
    }
    ```
  </Step>

  <Step title="Extract commit hashes">
    Runs `git log` to get commit hashes since the specified date:

    ```bash
    git -C "/path/to/repo" log --since="2026-03-03T00:00:00" --pretty=format:"%H"
    ```

    Limited to the 10 most recent commits to keep API requests manageable.
  </Step>

  <Step title="Collect commit diffs">
    For each commit hash, runs `git show` to get the full diff:

    ```bash
    git -C "/path/to/repo" show a79fc44 --unified=3 --pretty=format:"Commit: %H%nAuthor: %an%nDate: %ad%n"
    ```

    The diff includes commit metadata and code changes with 3 lines of context.
  </Step>

  <Step title="Build prompt">
    Combines all diffs into a structured prompt for Gemini:

    ```javascript
    function buildPromptFromDiffs(diffs) {
      const combinedDiffs = diffs.join("\n\n---\n\n");
      return `You are a 10x developer assistant.
      
      Input: Recent git commit diffs below:
      
      ${combinedDiffs}
      
      Task 1: Produce a "Developer Work Log" in markdown format...
      Task 2: Produce a "Manager Summary" with concise, non-technical bullet points...
      
      Output both sections separated by a horizontal rule.`;
    }
    ```
  </Step>

  <Step title="Call Gemini CLI">
    Spawns the Gemini CLI process and pipes the prompt via stdin:

    ```javascript
    const child = spawn("gemini", ["--model=gemini-2.5-pro"], {
      stdio: ["pipe", "pipe", "inherit"],
    });
    child.stdin.write(promptText);
    child.stdin.end();
    ```
  </Step>

  <Step title="Save worklog">
    Writes the Gemini response to a dated Markdown file:

    ```javascript
    const date = new Date().toISOString().slice(0, 10);
    const outPath = path.join(repo, worklogFolderName, `${date}.md`);
    fs.writeFileSync(outPath, `# Work Log — ${date}\n\n${result.trim()}\n`);
    ```
  </Step>
</Steps>

## Troubleshooting

### No commits found

If you see "No commits found since [date]", check:
- You have commits in the specified date range: `git log --since="2026-03-03T00:00:00"`
- The active project path is correct: `mcp-worklog.js list-projects`
- You're generating from the right date (try `yesterday` instead of `today`)

### Gemini CLI errors

If Gemini fails to respond:
- Verify Gemini is installed: `gemini --help`
- Check authentication: `gemini -p "test"`
- Review the spawn configuration in `mcp-worklog.js:147-161`

### Large repository issues

For repos with many commits or large diffs:
- The script limits to 10 commits automatically
- Increase `maxBuffer` in `mcp-worklog.js:44` if you see buffer errors:
  ```javascript
  { maxBuffer: 20 * 1024 * 1024 }  // 20MB
  ```

### Permission errors

If you can't write worklogs:
- Check the project directory exists and is writable
- Verify the path in `projects.json` is correct
- Ensure `worklogFolderName` doesn't conflict with existing files
